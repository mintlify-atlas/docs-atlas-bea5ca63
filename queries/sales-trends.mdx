## Overview

Time-based sales analysis is crucial for understanding business performance, identifying seasonal patterns, and forecasting future trends. This guide provides query patterns for analyzing sales data across different time dimensions.

## Sales Fact Table Fields

The `gold.fact_sales` table contains:

- `order_number` - Unique order identifier
- `product_key` - Foreign key to dim_product
- `customer_key` - Foreign key to dim_customer
- `order_date` - Date the order was placed
- `shipphing_date` - Date the order was shipped
- `sales_amount` - Total sales value
- `quantity` - Number of units sold
- `price` - Unit price

## Daily Sales Metrics

### Daily Sales Summary

```sql Daily Sales Overview
SELECT 
    order_date,
    COUNT(DISTINCT order_number) as orders,
    COUNT(DISTINCT customer_key) as unique_customers,
    SUM(sales_amount) as total_sales,
    SUM(quantity) as units_sold,
    AVG(sales_amount) as avg_order_value,
    AVG(price) as avg_unit_price
FROM gold.fact_sales
WHERE order_date >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY order_date
ORDER BY order_date DESC;
```

### Day-of-Week Analysis

```sql Sales by Day of Week
SELECT 
    TO_CHAR(order_date, 'Day') as day_of_week,
    EXTRACT(DOW FROM order_date) as day_number,
    COUNT(DISTINCT order_number) as total_orders,
    SUM(sales_amount) as total_sales,
    AVG(sales_amount) as avg_order_value,
    SUM(quantity) as units_sold
FROM gold.fact_sales
GROUP BY TO_CHAR(order_date, 'Day'), EXTRACT(DOW FROM order_date)
ORDER BY day_number;
```

<Tip>
Day-of-week patterns can inform staffing decisions, promotional timing, and inventory planning.
</Tip>

## Weekly and Monthly Trends

### Weekly Sales Performance

<CodeGroup>

```sql Weekly Sales Aggregation
SELECT 
    DATE_TRUNC('week', order_date) as week_start,
    COUNT(DISTINCT order_number) as orders,
    COUNT(DISTINCT customer_key) as customers,
    SUM(sales_amount) as weekly_sales,
    SUM(quantity) as units_sold,
    AVG(sales_amount) as avg_order_value
FROM gold.fact_sales
WHERE order_date >= CURRENT_DATE - INTERVAL '12 weeks'
GROUP BY DATE_TRUNC('week', order_date)
ORDER BY week_start DESC;
```

```sql Week-over-Week Growth
WITH weekly_sales AS (
    SELECT 
        DATE_TRUNC('week', order_date) as week_start,
        SUM(sales_amount) as weekly_revenue,
        COUNT(DISTINCT order_number) as weekly_orders
    FROM gold.fact_sales
    GROUP BY DATE_TRUNC('week', order_date)
)
SELECT 
    week_start,
    weekly_revenue,
    weekly_orders,
    LAG(weekly_revenue) OVER (ORDER BY week_start) as prev_week_revenue,
    weekly_revenue - LAG(weekly_revenue) OVER (ORDER BY week_start) as revenue_change,
    ROUND((weekly_revenue - LAG(weekly_revenue) OVER (ORDER BY week_start)) / 
          NULLIF(LAG(weekly_revenue) OVER (ORDER BY week_start), 0) * 100, 2) as growth_pct
FROM weekly_sales
ORDER BY week_start DESC
LIMIT 26;  -- Last 6 months
```

</CodeGroup>

### Monthly Sales Trends

```sql Monthly Sales Performance
SELECT 
    DATE_TRUNC('month', order_date) as month,
    TO_CHAR(order_date, 'Month YYYY') as month_name,
    COUNT(DISTINCT order_number) as orders,
    COUNT(DISTINCT customer_key) as unique_customers,
    SUM(sales_amount) as monthly_revenue,
    SUM(quantity) as units_sold,
    AVG(sales_amount) as avg_order_value,
    ROUND(SUM(sales_amount) / COUNT(DISTINCT customer_key), 2) as revenue_per_customer
FROM gold.fact_sales
GROUP BY DATE_TRUNC('month', order_date), TO_CHAR(order_date, 'Month YYYY')
ORDER BY month DESC;
```

### Month-over-Month Comparison

```sql Month-over-Month Growth Analysis
WITH monthly_metrics AS (
    SELECT 
        DATE_TRUNC('month', order_date) as month,
        SUM(sales_amount) as revenue,
        COUNT(DISTINCT order_number) as orders,
        COUNT(DISTINCT customer_key) as customers,
        SUM(quantity) as units
    FROM gold.fact_sales
    GROUP BY DATE_TRUNC('month', order_date)
)
SELECT 
    TO_CHAR(month, 'Month YYYY') as month_name,
    revenue as current_revenue,
    LAG(revenue, 1) OVER (ORDER BY month) as prev_month_revenue,
    ROUND((revenue - LAG(revenue, 1) OVER (ORDER BY month)) / 
          NULLIF(LAG(revenue, 1) OVER (ORDER BY month), 0) * 100, 2) as mom_growth_pct,
    orders as current_orders,
    LAG(orders, 1) OVER (ORDER BY month) as prev_month_orders,
    customers as current_customers,
    LAG(customers, 1) OVER (ORDER BY month) as prev_month_customers
FROM monthly_metrics
ORDER BY month DESC
LIMIT 12;
```

## Quarterly and Annual Analysis

### Quarterly Sales Performance

```sql Quarterly Sales Summary
SELECT 
    EXTRACT(YEAR FROM order_date) as year,
    EXTRACT(QUARTER FROM order_date) as quarter,
    'Q' || EXTRACT(QUARTER FROM order_date) || ' ' || EXTRACT(YEAR FROM order_date) as quarter_label,
    COUNT(DISTINCT order_number) as orders,
    COUNT(DISTINCT customer_key) as customers,
    SUM(sales_amount) as quarterly_revenue,
    SUM(quantity) as units_sold,
    AVG(sales_amount) as avg_order_value
FROM gold.fact_sales
GROUP BY EXTRACT(YEAR FROM order_date), EXTRACT(QUARTER FROM order_date)
ORDER BY year DESC, quarter DESC;
```

### Year-over-Year Comparison

```sql YoY Sales Comparison
WITH yearly_sales AS (
    SELECT 
        EXTRACT(YEAR FROM order_date) as year,
        SUM(sales_amount) as annual_revenue,
        COUNT(DISTINCT order_number) as annual_orders,
        COUNT(DISTINCT customer_key) as annual_customers
    FROM gold.fact_sales
    GROUP BY EXTRACT(YEAR FROM order_date)
)
SELECT 
    year as current_year,
    annual_revenue,
    LAG(annual_revenue) OVER (ORDER BY year) as prev_year_revenue,
    annual_revenue - LAG(annual_revenue) OVER (ORDER BY year) as revenue_change,
    ROUND((annual_revenue - LAG(annual_revenue) OVER (ORDER BY year)) / 
          NULLIF(LAG(annual_revenue) OVER (ORDER BY year), 0) * 100, 2) as yoy_growth_pct,
    annual_orders,
    annual_customers
FROM yearly_sales
ORDER BY year DESC;
```

## Category Sales Trends

### Category Performance Over Time

```sql Monthly Sales by Category
SELECT 
    DATE_TRUNC('month', f.order_date) as month,
    p.category,
    COUNT(DISTINCT f.order_number) as orders,
    SUM(f.sales_amount) as revenue,
    SUM(f.quantity) as units_sold,
    ROUND(SUM(f.sales_amount) * 100.0 / 
          SUM(SUM(f.sales_amount)) OVER (PARTITION BY DATE_TRUNC('month', f.order_date)), 2) as pct_of_monthly_sales
FROM gold.fact_sales f
INNER JOIN gold.dim_product p ON f.product_key = p.product_key
GROUP BY DATE_TRUNC('month', f.order_date), p.category
ORDER BY month DESC, revenue DESC;
```

### Category Trends Analysis

```sql Category Growth Trends
WITH category_monthly AS (
    SELECT 
        DATE_TRUNC('month', f.order_date) as month,
        p.category,
        SUM(f.sales_amount) as monthly_revenue
    FROM gold.fact_sales f
    INNER JOIN gold.dim_product p ON f.product_key = p.product_key
    GROUP BY DATE_TRUNC('month', f.order_date), p.category
)
SELECT 
    month,
    category,
    monthly_revenue,
    LAG(monthly_revenue) OVER (PARTITION BY category ORDER BY month) as prev_month_revenue,
    ROUND((monthly_revenue - LAG(monthly_revenue) OVER (PARTITION BY category ORDER BY month)) / 
          NULLIF(LAG(monthly_revenue) OVER (PARTITION BY category ORDER BY month), 0) * 100, 2) as mom_growth_pct,
    AVG(monthly_revenue) OVER (PARTITION BY category ORDER BY month ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) as three_month_avg
FROM category_monthly
ORDER BY category, month DESC;
```

## Customer Segment Trends

<Accordion title="Sales Trends by Customer Demographics">

```sql Monthly Sales by Demographics
SELECT 
    DATE_TRUNC('month', f.order_date) as month,
    c.gender,
    c.marital_status,
    COUNT(DISTINCT f.order_number) as orders,
    COUNT(DISTINCT f.customer_key) as customers,
    SUM(f.sales_amount) as revenue,
    AVG(f.sales_amount) as avg_order_value
FROM gold.fact_sales f
INNER JOIN gold.dim_customer c ON f.customer_key = c.customer_key
GROUP BY DATE_TRUNC('month', f.order_date), c.gender, c.marital_status
ORDER BY month DESC, revenue DESC;
```

</Accordion>

<Accordion title="New vs Returning Customer Trends">

```sql New vs Returning Customer Sales
WITH customer_first_order AS (
    SELECT 
        customer_key,
        MIN(order_date) as first_order_date
    FROM gold.fact_sales
    GROUP BY customer_key
),
order_classification AS (
    SELECT 
        f.*,
        CASE 
            WHEN DATE_TRUNC('month', f.order_date) = DATE_TRUNC('month', cfo.first_order_date) 
            THEN 'New Customer'
            ELSE 'Returning Customer'
        END as customer_type
    FROM gold.fact_sales f
    INNER JOIN customer_first_order cfo ON f.customer_key = cfo.customer_key
)
SELECT 
    DATE_TRUNC('month', order_date) as month,
    customer_type,
    COUNT(DISTINCT order_number) as orders,
    COUNT(DISTINCT customer_key) as customers,
    SUM(sales_amount) as revenue,
    AVG(sales_amount) as avg_order_value
FROM order_classification
GROUP BY DATE_TRUNC('month', order_date), customer_type
ORDER BY month DESC, customer_type;
```

</Accordion>

## Seasonal Analysis

### Seasonality Patterns

```sql Sales by Month of Year (Seasonal Pattern)
SELECT 
    EXTRACT(MONTH FROM order_date) as month_number,
    TO_CHAR(order_date, 'Month') as month_name,
    EXTRACT(YEAR FROM order_date) as year,
    COUNT(DISTINCT order_number) as orders,
    SUM(sales_amount) as revenue,
    AVG(sales_amount) as avg_order_value
FROM gold.fact_sales
GROUP BY EXTRACT(MONTH FROM order_date), TO_CHAR(order_date, 'Month'), EXTRACT(YEAR FROM order_date)
ORDER BY year DESC, month_number;
```

### Holiday Period Analysis

```sql Custom Period Comparison (e.g., Holiday Season)
SELECT 
    EXTRACT(YEAR FROM order_date) as year,
    CASE 
        WHEN EXTRACT(MONTH FROM order_date) IN (11, 12) THEN 'Holiday Season'
        ELSE 'Regular Period'
    END as period_type,
    COUNT(DISTINCT order_number) as orders,
    SUM(sales_amount) as revenue,
    AVG(sales_amount) as avg_order_value,
    COUNT(DISTINCT customer_key) as customers
FROM gold.fact_sales
GROUP BY EXTRACT(YEAR FROM order_date), period_type
ORDER BY year DESC, period_type;
```

## Fulfillment Metrics

### Order-to-Ship Time Analysis

```sql Shipping Performance Trends
SELECT 
    DATE_TRUNC('month', order_date) as month,
    COUNT(DISTINCT order_number) as total_orders,
    AVG(shipphing_date - order_date) as avg_days_to_ship,
    MIN(shipphing_date - order_date) as min_days_to_ship,
    MAX(shipphing_date - order_date) as max_days_to_ship,
    PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY shipphing_date - order_date) as median_days_to_ship
FROM gold.fact_sales
WHERE shipphing_date IS NOT NULL
GROUP BY DATE_TRUNC('month', order_date)
ORDER BY month DESC;
```

<Warning>
Long shipping times may impact customer satisfaction. Monitor this metric regularly and investigate anomalies.
</Warning>

## Rolling Averages and Moving Metrics

```sql 7-Day and 30-Day Moving Averages
WITH daily_sales AS (
    SELECT 
        order_date,
        SUM(sales_amount) as daily_revenue,
        COUNT(DISTINCT order_number) as daily_orders
    FROM gold.fact_sales
    GROUP BY order_date
)
SELECT 
    order_date,
    daily_revenue,
    daily_orders,
    ROUND(AVG(daily_revenue) OVER (ORDER BY order_date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW), 2) as moving_avg_7day,
    ROUND(AVG(daily_revenue) OVER (ORDER BY order_date ROWS BETWEEN 29 PRECEDING AND CURRENT ROW), 2) as moving_avg_30day,
    ROUND(AVG(daily_orders) OVER (ORDER BY order_date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW), 1) as moving_avg_orders_7day
FROM daily_sales
ORDER BY order_date DESC
LIMIT 90;
```

<Note>
Moving averages smooth out daily fluctuations and help identify underlying trends in the data.
</Note>
